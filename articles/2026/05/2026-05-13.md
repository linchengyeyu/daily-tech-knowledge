# 腾讯刚开源了个项目，一张照片直接变3D模型，SIGGRAPH都收了

昨天刷GitHub，看到一个让我愣了一下的项目。

腾讯ARC实验室出的，叫Pixal3D。

干嘛的呢？你给它一张普通照片，它直接给你生成一个3D模型。

不是那种粗糙的、歪歪扭扭的3D，是那种能直接拿来用的，带材质、带纹理、几何细节都在的高精度3D模型。

我当时第一反应是：这玩意不会又是PPT吧？

结果往下翻，看到了一行字——**SIGGRAPH 2026**。

做图形学的人应该知道这四个字什么分量。SIGGRAPH是全球计算机图形学顶会，能被收录基本上就是"这个方向做到头了"的意思。

而且这项目才开源3天，已经500多Star了。

你可能会问：AI生成3D不是早就有了吗？是的，但之前那些效果我都懒得说——你自己去跑一个试试就知道了，出来的东西跟原图的关系，大概就是"卖家秀"和"买家秀"的区别。

## 这个项目到底牛在哪？

简单说三个点。

**第一，"像素级对齐"。**

以前的AI生成3D模型，说白了是靠"猜"——AI看了你的照片，脑子里大概想象一个3D形状，然后生成出来。结果就是生成的模型跟原图总是差那么点意思，细节对不上。

Pixal3D换了个思路。它不是猜，它是直接把照片里每个像素的信息"投射"到3D空间里。就像你用投影仪把照片打到一个雕塑上，每一块颜色、每一条纹理都精准对应。

说大白话：以前的方案像你跟朋友说"帮我画个我家的猫"，朋友凭记忆画了一个，大概像但细节不对。Pixal3D像你直接把猫的照片贴在粘土上，照着捏，每个花纹都对。

**第二，清华+腾讯联手。**

作者名单里，清华大学、腾讯ARC实验室、新西兰维多利亚大学。

清华的胡事民教授是通讯作者，做图形学的应该都知道他。腾讯那边是ARC实验室，之前做过一堆AI图像方向的经典项目。

这种组合出来的东西，质量不会差。

说到这你大概明白了——这不是什么个人开发者的周末项目，是正儿八经的顶级研究成果。但让我意外的是，他们居然把代码完全开源了，还给了在线Demo。

**第三，真的能跑起来。**

这不是一个"论文放出来了但代码跑不了"的项目。他们直接在Hugging Face上放了一个在线Demo，浏览器打开就能用，不用装任何东西。

上传一张图片，等一会儿，3D模型就出来了。支持导出GLB格式，直接扔进Blender或者其他3D软件里接着改。

我试了一下，传了一张我桌上的手办照片，出来的3D模型把手办的轮廓、颜色甚至表面反光都还原了。虽然不能说100%一模一样，但比我预期的好太多了。

你要知道，这个效果放在一年前，得花好几个小时手工建模才能做到。现在几秒钟就出来了。

这东西谁会用？我想到几个场景：

做独立游戏的，拍个实物照片直接变3D素材，省了多少建模时间。做电商的，商品拍照就能出3D展示模型，比平面照片有吸引力多了。做短视频的，一个3D转场效果，门槛从"会Blender"降到"会拍照"。

## 说说限制

好的说完了，泼点冷水。

**安装是真费劲。**

Pixal3D底层依赖微软的TRELLIS.2框架，这个框架的安装过程……怎么说呢，GitHub Issues里已经有人在骂了。

有人直接说"TRELLIS.2的安装practically never works"（基本上从来没成功过），还有人要求他们支持UV环境管理工具，因为手动装环境"尤其在Windows上基本不可能"。

我看了看安装步骤，确实不是小白友好的。你得先装好TRELLIS.2的完整环境，再装Pixal3D的额外依赖，中间任何一步出错都得从头排查。

所以如果你想本地跑，做好折腾半天的准备。

**只有Python版本，没有ComfyUI插件。**

Issues里呼声最高的就是"能不能做个ComfyUI支持"。做AI绘画的人应该都懂ComfyUI，它是一个可视化工作流工具，拖拽就能用。Pixal3D目前只有命令行和Gradio Web界面，想集成到ComfyUI工作流里还得自己写节点。

**需要显卡。**

3D生成这个事，没有一张像样的GPU是不行的。官方没说最低显存要求，但TRELLIS.2框架建议8G以上。所以如果你的电脑是核显或者老显卡，本地跑基本没戏。

好在有在线Demo可以白嫖。

## 怎么用？

两个选择：

**简单路线（推荐）：** 直接去Hugging Face Demo，浏览器里打开，上传图片，等着就行。

地址：huggingface.co/spaces/TencentARC/Pixal3D

**硬核路线：** 本地安装。前提是你有一张8G以上显存的NVIDIA显卡，并且有耐心。

```bash
# 先装TRELLIS.2（这一步最折腾）
# 按照微软的官方文档来

# 然后装Pixal3D的额外依赖
pip install -r requirements.txt
pip install https://github.com/LDYang694/Storages/releases/download/20260430/utils3d-0.0.2-py3-none-any.whl

# 生成3D模型
python inference.py --image 你的图片.png --output 输出.glb
```

说实话，对于大多数人来说，用在线Demo就够了。

## 我的想法

我不是搞3D的，我就是个做安全工作的普通人。但这个项目让我觉得有意思的地方在于——它说明AI正在把越来越多"只有专业团队才能做的事"变成"普通人也能碰的事"。

以前做一个3D模型，你得学Blender、学建模、学贴图，没个半年入不了门。现在你拍张照片，等几秒钟，3D模型就有了。

当然，现在的精度离专业级还差那么一截。但方向是对的。

腾讯开源这个项目的时机也挺微妙。SIGGRAPH 2026的论文、微软TRELLIS.2的底层支持、清华的技术背书——这配置摆出来，就是告诉所有人：3D AI生成这个赛道，我们认真了。

等哪天安装过程能简化到"一键部署"，那才是真正的全民3D时代。
