## 一、一个 ImportError，让所有 Cron 任务集体罢工

**场景**：我有个 AI Agent（Hermes），每天早上6点和9点各跑一个定时任务——一个回收昨天的对话提取知识点，一个抓取AI新闻生成日报。两个任务已经稳定运行了一个多星期。

但5月1日早上，两个任务同时失败了。报错信息一模一样：

```
Error processing job: cannot import name 'cfg_get' from 'hermes_cli.config'
```

**排查过程**：

1. 先检查 Cron 日志目录——5月1日根本没有输出文件，说明任务在初始化阶段就崩了
2. 翻 `agent.log`，找到了确切报错行
3. 回忆时间线：4月30日晚 gateway 做过一次重启（因为代码升级），重启后加载的新版本代码里 `cfg_get` 函数被移除或重命名了
4. 所有依赖这个函数的模块全部受影响，不仅是 Cron，连普通对话都报错

**解决方案**：重启 gateway（`hermes gateway restart`），让它加载修正后的代码版本。

**教训**：

- 🔑 **代码升级后一定要验证定时任务**。别以为升级完了、主流程通了就没事，Cron 任务用的是同一套代码，一个函数删了就是全局故障
- 🔑 **`agent.log` 是排查 Cron 失败的金矿**。Cron 任务没有交互界面，日志文件几乎是唯一的诊断手段
- 🔑 **升级后手动触发一次定时任务**，确认能正常跑通，再等第二天自动执行

## 二、公众号文章发布流程：AI不能"自己说了算"

之前我让 AI Agent 写完文章后，自己按八维度评分体系打个分，分够了就直接发到公众号草稿箱。听起来很自动化对吧？

**但用户（也就是我）不答应。**

问题出在：AI 觉得文章 88 分"够好了"，直接发了。但我作为运营者，有涨粉策略的考量——标题要不要加悬念、互动环节放在哪里、哪些内容更适合目标读者。这些 AI 不懂。

**修正后的流程**：

1. AI 写完文章 → 按八维度打分（选题、标题、结构、信息、语言、视觉、互动、合规）
2. **不管多少分，先交给我审核**（这是关键！）
3. 我看完给出修改方向
4. AI 按方向改完再评分
5. 达到 90 分以上才正式发布

**核心原则**：自动化不是自动决策。AI 可以承担 80% 的工作，但最后 20% 的判断——关乎策略、品牌调性、用户心理——必须由人来做。

**八维度评分实战数据**（AI世界日报评分）：

| 维度 | 得分 | 主要扣分原因 |
|------|------|------------|
| 标题吸引力 | 6/10 | 太 generic，缺乏悬念 |
| 互动引导 | 2/5 | 零互动设计 |
| 视觉呈现 | 7/10 | 无数据可视化 |
| **总分** | **88.5/100（B+级）** | |

你看，标题和互动这两项，AI 不会自动想到"这里加个提问能涨粉"。这就是人工审核的价值。

## 三、Hermes Agent v0.11.0 升级：5个合并冲突的生存指南

（注：这个知识点来自4月30日的升级实操，但教训是通用的。）

从 v0.10.0 升级到 v0.11.0 时，一口气遇到 5 个 merge conflict：

| 冲突文件 | 原因 |
|---------|------|
| `agent/anthropic_adapter.py` | DeepSeek 端点检测逻辑变了 |
| `gateway/run.py` | context length 解析方式重构了 |
| `hermes_cli/models.py` | 模型验证逻辑大改 |
| `run_agent.py` | 配置读取结构变了 |
| `tests/...` | 测试用例要跟着改 |

**关键经验**：

1. **先 diff 再打补丁**：上游可能已经集成了你的自定义修改。这次升级中，DeepSeek 的 Anthropic 兼容端点检测和 reasoning_content 处理就被上游吸收了，盲目重新打补丁只会制造冲突

2. **区分"上游已吸收"和"需要手动恢复"**：
   - ✅ 上游吸收的：DeepSeek 检测逻辑、reasoning_content 兼容 → 不用管
   - ❌ 需要手动恢复的：`max_tokens` 配置化读取、自定义 Provider 归一化 → 重新应用

3. **升级后立刻验证**：跑一下核心功能，确认没有回归。否则就像上面说的，第二天早上 Cron 就给你"惊喜"

## 总结

今天的三个教训可以归结为一句话：**自动化系统最脆弱的时刻，就是你改完代码之后的第一天。**

- Gateway 重启 → `cfg_get` 函数丢失 → Cron 全挂
- 升级后没验证 → 第二天才发现问题
- AI 自动发布 → 跳过人工审核 → 错过涨粉机会

自动化是为了节省时间，但**省下来的时间应该花在验证上，而不是省掉验证**。
