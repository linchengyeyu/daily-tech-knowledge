## 🎯 PPT Master：AI终于能做出「真正可编辑」的PPT了

如果你用过AI生成PPT的工具，一定遇到过这个尴尬：生成的PPT要么是图片拼的，要么导出后无法编辑。**PPT Master** 彻底解决了这个问题。

这个开源项目（GitHub: hugohe3/ppt-master）目前已经有 **10.7k Star**，核心卖点就一个：**生成的是原生PPTX文件，每个形状、文本框、图表都是独立可编辑对象**。你在PowerPoint里想怎么改就怎么改，跟手工做的一模一样。

**它的完整工作流是这样的**：

1. 你丢给它PDF、Word、URL或Markdown文档作为素材
2. AI进入"策略师"模式，跟你确认风格、配色、字体等8项参数
3. 逐页生成SVG矢量图，最后合成PPTX导出

最新版本v2.5.0还新增了**音频旁白、视频导出和页面转场动画**，功能越来越完整。

**踩坑经验**：安装时如果遇到 pycairo 编译失败，别慌，`brew install pkgconf` 就能解决（前提是系统已装好cairo）。建议用UV做Python环境隔离，干净又不会污染系统。

内置22个示例项目共309页，风格从杂志风到暗黑艺术风都有。对于经常要做技术分享、产品演示的朋友来说，这是一个值得收藏的工具。

## 🔥 VS Code Copilot强制署名，682分引爆HN

这可能是本周AI圈最有意思的争议了。

有开发者发现，**VS Code会在Git提交中自动添加"Co-Authored-by: Copilot"**——不管你这次提交有没有实际用到Copilot。这个行为在Hacker News上获得了682分讨论热度。

争议的核心在于**归属权和透明度**。如果你没用Copilot，它凭什么在commit里署名？如果用了，这种自动添加的方式是否合理？这背后涉及到一个更深的问题：**AI辅助开发的贡献到底该如何界定？**

目前社区看法两极分化：
- 一方认为这有助于追踪AI使用情况，提高代码来源透明度
- 另一方认为这是强制性的"AI标签"，侵犯了开发者的选择权

对于普通开发者来说，如果你在意commit history的干净程度，可以在VS Code设置里关闭这个功能。

## 📜 一键生成软件著作权申请材料

如果你在国内做过软著申请，一定知道那个痛苦的过程：整理代码文档、写说明书、准备各种表格。

**SoftwareCopyright-Skill**（GitHub: Fokkyp/SoftwareCopyright-Skill）这个开源工具直接帮你搞定。它读取你的本地项目代码，**自动生成全套.docx格式的软件著作权申请材料**，636 Star，完全开源免费。

对于独立开发者和创业团队来说，这简直是神器。软著申请虽然不难，但材料准备极其繁琐耗时间，能自动化就别手动。
