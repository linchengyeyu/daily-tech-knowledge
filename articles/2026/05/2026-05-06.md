## DeepClaude：一行命令让 Claude Code 用上 DeepSeek

你有没有觉得 Claude Code 很强，但太贵了？有个叫 **deepclaude** 的开源项目刚出来三天就拿了 1300 多 Star，做的事很简单——**把 Claude Code 的底层模型换成 DeepSeek V4 Pro，成本直接砍到原来的 1/17**。

怎么做到的？原理其实挺巧妙。Claude Code 启动时会读取 `ANTHROPIC_BASE_URL` 这个环境变量来决定 API 请求发到哪。deepclaude 就是把这个地址从 Anthropic 的服务器改成了 DeepSeek 的兼容端点。**你甚至不需要 Anthropic 账号**，一个 DeepSeek API Key 就够了。

装好之后终端输入 `deepclaude`，界面和操作跟正版 Claude Code 完全一样。它还支持运行中热切换后端——本地起了一个代理服务（localhost:3200），可以随时切到 OpenRouter 或 Fireworks AI。

**但要注意几个坑**：

- 如果你之前登录过 Claude 账号，要先用 `claude logout` 退出，否则请求可能还是走 Anthropic 的 token（这是 GitHub Issue #3 报的 bug）
- 不支持图片和视觉输入，纯文本编码场景没问题
- MCP 工具不兼容，复杂 Agent 工作流暂时用不了
- macOS 上 benchmark 命令有兼容性问题，不影响正常使用

实际效果怎么样？社区反馈说日常编码 80% 的场景和 Claude Opus 差距不大，复杂架构设计和大文件重构还是原版更强。

> 项目地址：GitHub 搜索 `aattaran/deepclaude`，MIT 协议免费使用

## 公众号冷启动踩坑：写给程序员看 = 没人看

我们自己的公众号「AI智管局」最近做了一次深度复盘，发现了一个很多技术号都会犯的错误——**你以为在分享干货，读者眼里全是天书**。

### 三大核心问题

1. **定位跑偏**：目标用户是普通小白，结果写的内容全是「Hermes Agent 大版本升级」「Gateway 重启干翻定时任务」这种程序员日记
2. **没有传播点**：标题「每日技术干货 | 5月4日」——你看到会点吗？
3. **单渠道零矩阵**：抖音、小红书、知乎一个都没开

### 拆解爆款号的套路

我们研究了几个百万粉级别的 AI 公众号，其中「逛逛GitHub」的套路特别值得学习：

- **标题必须有冲突**：「GitHub 上狂揽 1.1 万 Star，22 岁开发者逆向工程了 Claude」——数字 + 年龄反差 + 逆向工程，三个钩子一次拉满
- **一句一段**，手机上滑得飞快，完全没有阅读压力
- **图片用证据型截图**，不是花哨的设计图，直接贴 GitHub Issue 截图、代码截图，真实感拉满
- **故意写争议**，Issues 里有人骂有人夸的内容也放出来，评论区直接炸

核心思路就一条：**别写技术日记，写普通人能用 AI 干什么**。

## 今日小结

- DeepClaude 项目让 Claude Code 的成本降了 17 倍，日常编码完全够用，复杂场景还是上原版
- 公众号运营的关键不是写得多专业，而是让目标用户看得懂、愿意点、愿意转发
- 标题是第一生产力，没有冲突和悬念的标题就是在浪费好内容
