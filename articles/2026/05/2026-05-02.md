## 一、Karpathy：未来写 .md 就够了，别写 .sh 了

前两天，Karpathy 在红杉的 Ascent 大会上做了一个分享，提出了三个让我眼前一亮的观点。

**第一个：菜单驱动的软件要完了。**

你想想，以前我们用软件，先找菜单，再找功能，找到之后还要填一堆表单。现在呢？直接告诉 AI 你要什么，它就给你做了。这事儿正在发生，不是预测。

**第二个：Markdown 文件会替代 Shell 脚本。**

这个观点让我特别有共鸣——因为我每天都在用 Hermes Agent 的 Skill 系统，每个 Skill 就是一个 `.md` 文件，用自然语言描述工作流程。说实话，比写 shell 脚本直观太多了，改起来也方便。Karpathy 站在更高的层面说了同一件事。

🤔 **你平时写自动化脚本多还是用 AI Agent 多？评论区聊聊**

**第三个：非结构化知识库变得"可计算"了。**

以前要把知识塞进数据库，还得建表、定义 schema、写 SQL。现在？直接扔一堆 Markdown 文件给 LLM，它就能理解、检索、推理。我自己就在用 Obsidian 管理知识库，配合 AI 搜索，确实比传统方式灵活太多了。

他还解释了一个有趣的现象——为什么 AI 在编程和写作上很强，但在一些"看起来很简单"的事情上反而拉胯？**因为训练数据的分布不均匀，本质上是个经济问题**，不是技术缺陷。被讨论得多的领域（编程、写作）数据多，模型就强；冷门领域数据少，模型就弱。

## 二、Claude Security：不用搭环境就能审代码漏洞

Anthropic 放了个大招——**Claude Security 公开测试版**上线了。

简单说，它让你不需要搞什么安全扫描工具，直接指向一个代码仓库，就能找出漏洞，而且在同一个界面里就能修复。**目前已有数百家公司在生产环境代码上使用。**

几个关键信息：
- 基于 Claude Opus 4.7 模型构建
- 支持定时扫描、Webhook 告警、CSV 导出
- 已集成到 Claude Code 网页版
- 支持目录级扫描，适合大型项目

以前搞代码安全审计得配 SAST 工具、写规则、调参数，现在变成"给个仓库地址就行"。对于小团队和独立开发者来说，这简直是降维打击。

## 三、Open Design：11.8k Star 的开源设计全家桶

GitHub 上冒出来一个爆款——**Open Design**（nexu-io/open-design），上线就拿到了 **11,800+ Star**。

它的核心卖点：**内置 71 种品牌级设计系统**（Stripe、Linear、Vercel 风格应有尽有），可以生成网页原型、桌面界面、移动端设计、幻灯片、图片甚至视频。而且兼容 Claude Code、Codex、Cursor、Gemini 等主流 AI Agent。

对于"审美一般但需求很多"的开发者来说，太香了。以前用 Figma 调半天还调不好，现在用自然语言描述风格，AI 帮你生成。

最关键的是——**完全开源**，本地运行，数据不上传。

## 四、Codex 升级：不写代码也能用了

Sam Altman 连发几条推，核心就一句：**Codex 大升级，日常电脑操作也能用它了。**

他鼓励大家试试用 Codex 做非编码任务——整理文件、操作应用、自动化工作流。AI Agent 能力边界正从"编程助手"扩展到"通用电脑助手"。

OpenClaw 的 Peter Steinberger 也站台：**Codex 比 GPT 引擎好使，而且不会乱生成 Markdown 文件**——这对用 AI 写代码的人来说是个很实际的优点。
