## 一、Hermes Agent 大版本升级：1614 个提交的跨越

Hermes Agent 是 Nous Research 开源的 AI Agent 平台，我们日常用它的 Skill 系统、Cron 定时任务、多平台消息分发等功能来管理公众号和知识库。

这次从 v0.10.0 升级到 v0.11.0（上游标签 v2026.4.23），一口气合并了 **1614 个提交**。听起来很爽，但实际操作中遇到了 **5 个合并冲突**——都是之前为了兼容 DeepSeek 而做的本地修改。

### 冲突怎么解？

升级脚本自动把冲突文件重置为上游状态，本地改动被 `git stash` 保存。关键是 **别急着把 stash 的内容全部重新应用**——上游可能已经包含了你的部分修改。

实际操作分三类：

| 类型 | 文件 | 处理方式 |
|------|------|----------|
| 上游已集成 | `agent/anthropic_adapter.py` | DeepSeek 协议检测 + thinking block 处理，上游已经内置了，不需要再改 |
| 上游已集成 | `gateway/run.py` | 上下文长度显示改用了更好的 `resolve_display_context_length` 方案 |
| 需重新应用 | `run_agent.py` | 自定义 `max_tokens` 从 config.yaml 读取的功能，上游还没加上 |
| 需重新应用 | `hermes_cli/models.py` | `custom:DeepSeek` 命名的自定义 Provider 归一化 |

### 升级经验总结

> **先 diff，再 patch** —— 不要盲目把所有 stash 内容重新应用，逐个对比上游代码是否已包含你的修改。

这次升级后确认了两件事：
- DeepSeek 的 thinking block 兼容问题已经在上游代码中解决了，之前我们的源码 patch 可以安全移除
- `max_tokens` 自定义配置读取功能依然需要手动维护，已经提交 PR 到上游但尚未合并

## 二、公众号文章质量门禁：八维度评分体系

之前公众号文章的发布流程是：AI 写完 → 直接推草稿箱 → 人工检查后发布。

问题很明显：**AI 对自己的文章质量缺乏判断力**。标题够不够吸引？互动引导有没有？配图合不合适？AI 自己很难客观评价。

所以我们设计了一套 **八维度评分体系**，让 AI 写完文章后先用 DeepSeek API 自评打分：

| 维度 | 权重 | 考察什么 |
|------|------|----------|
| 选题价值 | 20% | 切中用户痛点，传播潜力 |
| 标题与封面 | 10% | 吸引点击 |
| 内容结构 | 15% | 逻辑清晰 |
| 信息质量 | 20% | 准确、有深度 |
| 语言表达 | 10% | 流畅、有风格 |
| 视觉呈现 | 10% | 配图和排版 |
| 互动引导 | 5% | 与读者的互动 |
| 合规与价值观 | 10% | 内容安全 |

**总分 ≥ 90 才能发布**，否则按扣分维度自动修改后重新评分。

实测效果：4月30日首次运行，文章得分 **88 分**，差 2 分没达标。扣分集中在「互动引导」（7/10）和「标题与封面」（8/10）。修改后重新评分达到 92 分，顺利进入草稿箱。

这套评分体系现在已经集成到每日自动化的 Cron 任务中，每天早上 6 点回收知识、写文章、评分、发布，全程无人值守。

## 三、delegate_task 中断：AI 子任务的"翻车"时刻

在调试文章评分的过程中，我们发现了一个值得注意的问题：**AI 的子任务（delegate_task）可以被用户的下一条消息中断**，导致所有工作丢失。

具体场景：AI 派了一个子任务去历史对话中搜索文章评分记录，子任务跑了近 30 分钟（16 次 session_search 调用），然后用户发了一条新消息——子任务立刻被中断，所有搜索结果全部丢失。

**教训**：对于需要大量搜索和处理的工作，不要把所有逻辑放在一个子任务里。更好的做法是分步执行，每步产出中间结果保存到文件，即使被中断也能从断点继续。

这个经验对所有使用 AI Agent 并发子任务的开发者都有参考价值——**子任务设计要有断点续传能力**。

## 四、bm.md 流程图上传：微信图床的正确用法

在给公众号文章添加流程图时，遇到了一个常见问题：**bm.md 渲染的 Markdown 中，本地图片路径在微信里打不开**。

正确流程是：
1. 先把流程图渲染成图片（PNG）
2. 通过微信 `media/uploadimg` API 上传到微信素材库
3. 获取微信图床 URL
4. 把 URL 嵌入到 Markdown 中
5. 再用 bm.md 渲染成公众号 HTML

**关键点**：微信素材库的图片 URL 和普通网页图片不同，必须通过 API 上传才能在公众号文章中正常显示。直接使用外部链接会被微信拦截或压缩。
